# Proposed README Structure

1. **Header**
   - Project title, one-line description, key badges (license, PRs welcome, Buy Me a Coffee).

2. **Why Meta Manager Config**
   - Short explanation of what the repo offers plus screenshot collage link.

3. **Quick Start**
   - Minimal Kometa config snippet for a single sport.
   - Links to docs for prerequisites and Kometa install.

4. **Repository Map**
   - Table/list describing `metadata/`, `scripts/`, `images/`, `posters/`, `overlays/`, `templates/`, `docs/`.

5. **Supported Sports Overview**
   - Table with columns: Sport, Seasons, Assets (metadata/posters/title cards), Guide link.
   - Badges/emoji to indicate completeness or “in progress”.

6. **Sport Guides**
   - One-paragraph summary that points to each `docs/*.md`.
   - Move descriptive marketing copy + screenshots from README into each guide.

7. **Movies & TV Enhancements**
   - Keep existing collections/overlay configs but summarize and link to deeper docs (future) or anchor subsections.

8. **Automation & Scripts**
   - Brief section listing generator scripts, pointing to `docs/scripts.md` (new) for usage details.

9. **Contributing**
   - Short invitation referencing `docs/contributing.md`, poster guidelines, templates.

10. **Templates & Assets**
   - Link to `templates/` inventory plus quick instructions for downloading.

11. **Support**
   - Buy Me a Coffee link and contact/issue guidance.
